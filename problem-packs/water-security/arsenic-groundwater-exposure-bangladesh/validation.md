# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Upazila-level exposure baselines must document:

- Upazila identifier and administrative boundary version.
- Arsenic measurement method (field kit, laboratory AAS, ICP-MS).
- Well depth stratum.
- Number of wells tested per upazila.
- Population denominator source.
- Missingness handling for untested wells.
- Administrative-boundary changes since the BGS/DPHE survey.

## Model Or Signal Requirements

No testing-coverage gap claim may be called actionable until it includes:

- Input data sources and their dates.
- Geographic grain (upazila or union).
- Testing-coverage estimate and its uncertainty.
- Exposure-overlap calculation method.
- Known non-use cases.
- Replication status.
