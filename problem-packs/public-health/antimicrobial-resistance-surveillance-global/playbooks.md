# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. Keep quantitative outputs reproducible.
5. Mark every operational inference as reviewed, replicated, or blocked.

## Reviewer Playbook

Ask five questions:

1. What surveillance-gap claim is being made?
2. What GLASS, lab, and burden data supports it?
3. What method produced the gap estimate?
4. What would make it false?
5. Who could misuse a gap map to justify restricted antibiotic access rather than lab investment?

## Field-Reality Playbook

Do not ask whether a gap map is impressive. Ask whether it changes a named lab-network planning decision for a named user at a useful time, without creating worse incentives.
