# Validation

## Validation Layers

| Layer           | Gate                                                                                           |
| --------------- | ---------------------------------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks                            |
| Evidence        | Every claim has a dated evidence record with product category, test method, and sampling frame |
| Reproducibility | Quantitative artifacts include rerunnable steps                                                |
| Review          | Required reviewer roles approve or request changes                                             |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced                           |

## Failure-Mode Requirements

Every accepted claim must state whether it refers to:

- Substandard products (quality failure from manufacturing, degradation, or storage)
- Falsified products (deliberate fraudulent mislabeling of identity or source)
- An aggregate that conflates both (permitted only for framing, not for product-level or geographic claims)

Claims that conflate both without stating so are rejected.

## Surveillance-Gap Requirements

No surveillance-gap map may be called actionable until it includes:

- Surveillance-density metric (laboratories or testing events per unit population or market volume)
- Market-volume denominator source
- Product categories tested versus untested
- Supply-chain tiers covered
- Geographic grain and its limits
- Known non-use cases
- Replication status
