# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. Never conflate substandard and falsified failure modes in a single claim.
5. Document sampling frame, test method, and geographic grain for every quality datum.
6. Keep quantitative outputs reproducible.

## Reviewer Playbook

Ask five questions:

1. What claim is being made about medicine quality?
2. What source proves it, and was the sampling frame random or convenience?
3. What test method was used, and does it match the failure mode claimed?
4. What would make it false — is there a surveillance gap being mistaken for a quality gap?
5. Who could misuse it — could this trigger a panic-driven withdrawal that reduces access?

## Field-Reality Playbook

Do not ask whether a surveillance-gap map is impressive. Ask whether it changes a named decision for a named regulator or procurement officer at a useful time, without reducing medicine access in the districts it identifies as untested.
